PCU FAULT RECOVERY REFERENCE

Panel placard — affixed to EPS Control console, port side.
PCU FAULT RECOVERY MODE — ENGINEER'S REFERENCE

  Trigger condition:   Measured load exceeds reported load by >15 MW
  Response:            PCU logs all channel data to passive buffer (immutable)
  PCU mode label:      FAULT RECOVERY
  Meaning:             The PCU itself is FUNCTIONING CORRECTLY.
                       The FAULT is in the DISTRIBUTION PATH — one or more
                       systems are drawing power while their status registers
                       report them as not drawing power.

  The system state that produces this condition:

    FAULT (●) — Hardware failure mode. A system in FAULT state may:
      - Draw power despite reporting no load
      - Report incorrect status to software registers
      - Operate partially or unexpectedly
      - Require physical inspection to determine actual operating condition

  FAULT ≠ OFFLINE. OFFLINE means no power (upstream dependency not met).
  A FAULT system can be drawing full rated load while its status reads OFFLINE.

  PCU MODE FLAG:        ● FAULT
