BEZEL PLACARD — ACCESS RESULT DEFINITIONS

Panel legend — visible on console bezel, port side.
ACCESS RESULT CODES
══════════════════════════════════════════════════════════════════════

  GRANTED   All three access gates cleared:
              (1) Console type authorised
              (2) Deck location authorised
              (3) User clearance authorised
            Access permitted through normal protocol.

  DENIED    One or more gates failed.
            Access refused. Entry not made. Badge event logged.

  FORCED    Access executed despite gate failure.
            A command-level override was applied to bypass the
            failed gate(s). Door opened. Entry made.
            FORCED access requires command-level authority — it
            cannot be triggered by the badge-holder alone.

══════════════════════════════════════════════════════════════════════
  NOTE: FORCED result means someone with override authority
  opened that door. The badge-holder did not defeat the system —
  the system was commanded to yield.

PHYSICAL IMPOSSIBILITY REFERENCE

Security protocol note — posted at all badge reader terminals.
PHYSICAL IMPOSSIBILITY FLAG — BADGE AUDIT PROTOCOL
══════════════════════════════════════════════════════════════════════

  RULE:  The same personnel ID cannot register swipe events in
         rooms 2 or more decks apart within 30 seconds.

  BASIS: Physical transit time. Deck-to-deck emergency transit
         (stairwells, emergency ladders) requires a minimum of
         approximately 18 seconds per deck under ideal conditions.
         More than 2 decks in under 30 seconds is not physically
         achievable.

  WHEN THIS PATTERN IS DETECTED:
    - The badge ID has been cloned, OR
    - The record has been falsified, OR
    - Both events involve the same access override operation

  In all cases: flag for Security review.
