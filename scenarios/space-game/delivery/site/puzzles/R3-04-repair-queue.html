BEZEL PLACARD — SYSTEM STATUS SYMBOLS

Reference panel affixed to damage control board, top edge. All engineering personnel are expected to recognise these states.
SYSTEM STATUS SYMBOLS — DAMAGE CONTROL REFERENCE
════════════════════════════════════════════════════════════════════════════

  ◉  ONLINE      System fully operational.

  ◇  READY       System initialised, standing by. All checks passed.
                 Awaiting activation command or upstream dependency.

  ◌  STANDBY     System powered but inactive. Repair or initialisation
                 in progress. Not yet ready for operations.

  ●  FAULT       System failure detected. Hardware or software fault
                 condition. Requires diagnosis and repair action.

  ⊘  LOCKED      System deliberately isolated from repair queue.
                 Does not progress to STANDBY. Does not progress to READY.
                 Does not progress to ONLINE. Remains isolated until
                 manual unlock command is issued.
                 LOCKED systems are not broken — they are held.

════════════════════════════════════════════════════════════════════════════

STANDARD RECOVERY SEQUENCE:

     ●  FAULT  →  ◌  STANDBY  →  ◇  READY  →  ◉  ONLINE

     Each step requires engineering action or automated repair protocol.
     Normal faults progress through the sequence within 1–3 snapshot
     intervals depending on severity.

ANOMALOUS SEQUENCE:

     ●  FAULT  →  ⊘  LOCKED

     A system goes directly from FAULT to LOCKED when a command-level
     isolation order is issued before repair can begin. This bypasses
     the recovery sequence entirely. The system will not self-repair.
     It will not appear in the automated repair queue.
     It will not return to service without a deliberate unlock.
